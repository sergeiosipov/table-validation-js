Batch Config Inference

Pick many table files (or a whole folder), infer one draft config per file, download everything as a ZIP. Files are read locally in this page — they never leave the machine; the only network access is cdn.jsdelivr.net. Drafts are suggestions, never authoritative — review them before use (the Authoring & Run Console is the place to do that).
1 · Pick files
Supported (mapped from the extension — nothing is content-sniffed): .csv · .tsv · .txt (read as CSV) · .xlsx · .json (an array of records ingests as jsonObjects, an array of rows as jsonArrays — the one stated peek this tool makes). A folder pick recurses; unsupported files are listed as skipped, never silently dropped.
2 · Options — applied to every file

“all accepting formats” drafts every accepting temporal format (union coverage for mixed-date columns) — usually what you want for messy feeds.
3 · Infer
The ZIP carries one <file>.config.json per success (engine-consumable, exactly what the console would export), optional <file>.report.json evidence, and a manifest.json naming every input with its outcome — failures included. The combined XLSX is the human review copy: one sheet per inferred file, each column annotated above the data with its inferred type, format, precision, nullability, confidence and sample evidence — the full ingested data below.
